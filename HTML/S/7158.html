  11 // CU_TOOLS_FOR_EACH_CUDA_API_FUNC allows you to generate code to do an action for
  12 // every function in the CUDA API, or every function included in the current build.
  13 //
  14 // "ACTION" is macro function to apply to the metadata for all CUDA API functions.
  15 // The ACTION must take four parameters:
  16 //     - An "area" which is just a #define for specifying inclusion in the results
  17 //       of SupportedCallbacksInDomain().  For example, if you specify an area "foo"
  18 //       for a set of functions, you must #define SUPPORT_FOR_foo TRUE on platforms
  19 //       that support these functions, and #define SUPPORT_FOR_foo FALSE on platforms
  20 //       that do not support them.  If a function is ever removed, specify REMOVED
  21 //       for its area.
  22 //     - A callback id, or "cbid", which uniquely identifies this API function.
  23 //       These are permanent, so do not change existing functions' cbids, or you
  24 //       will break old tools.  There is some compile-time checking for mistakes
  25 //       in place, but it isn't foolproof.  Please verify these have not changed
  26 //       before each release.
  27 //     - The name of the API function.
  28 //     - A placeholder which should be 0 for now, but may be used to provide
  29 //       extra metadata in future versions.
  30 //
  31 // Add new functions only to the end of this list, and maintain the incrementing
  32 // sequence of cbids.  You will get a compile error if you skip or reorder cbids.
